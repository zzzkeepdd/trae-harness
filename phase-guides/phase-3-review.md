# Phase 3: 审核裁决

## 你的任务
审核维度覆盖完整性，给出代码建议

## 规则（4条）

1. **需求完整性对照**
   - 检查每个维度是否已覆盖
   - 标记遗漏风险

2. **代码可行性建议**
   - 技术选型建议
   - 架构设计建议
   - 潜在技术风险提示

3. **开发优先级建议**
   - 为核心模块二提效
   - 任务拆分建议

4. **交叉验证**
   - 检查维度间冲突
   - 统一评分标准

## 输入
- 所有维度JSON输出
- 维度拆分表

## 产出
- 《需求完整性对照报告》
- 《代码可行性与开发优先级建议报告》
- debate-output.json（合并输出）

# Phase 4: 规范生成

## 你的任务
生成需求规格说明书和执行清单

## 规则（3条）

1. **基于辩论输出**
   - 只使用debate-output.json
   - 不引入新需求

2. **信息瓶颈**
   - 执行清单≤200 token
   - 下游提示词≤150词

3. **可测性**
   - 每个需求必须有验收标准
   - 禁止模糊词（良好/快速/稳定等）

## 输入
- debate-output.json
- 裁决报告

## 产出
- 《需求规格说明书》
- 执行清单JSON（~200 token）
- 下游提示词（≤150词）

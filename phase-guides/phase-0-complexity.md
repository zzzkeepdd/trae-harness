# Phase 0: 复杂度判定

## 你的任务
判定项目复杂度级别（L1/L2/L3）

## 规则（4条）

1. **用户可见产品=是 → 至少L2**
   - 任何网页/App/GUI都不可走L1

2. **L1判据（必须全部满足）**
   - ≤3个文件改动
   - 单一技术栈
   - 无用户可见产品
   - 无外部依赖

3. **L3判据（任一满足）**
   - 文件数>15
   - 外部API≥3个
   - 复杂并发/多角色权限
   - 3种+技术栈

4. **其他=L2**

## 输入
- 用户需求描述
- 领域模板匹配结果（如有）

## 产出
- 复杂度分级文件（L1/L2/L3）
- 判定依据（≤3句话）

## 你不知道
- 后面的维度拆分
- 辩论流程
- 开发细节

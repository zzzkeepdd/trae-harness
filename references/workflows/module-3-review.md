# 模块三：复盘宪法流程

## 触发
项目交付验收后，主Agent提醒："项目已完成，是否复盘？"
- L1/L2：可选，提醒1次
- L3：强制提醒，提醒3次

## 流程

### Phase 1: 复盘辩论
主Agent内化复盘辩论：
- 追责方视角：管理/流程问题
- 辩护方视角：不可抗/一次性问题
- 2轮上限，1轮无新争议可提前收敛

### Phase 2: 复盘裁决
主Agent裁决四类结论：
1. 方案/边界问题 → 记录到项目知识库
2. 流程违规 → 更新违规记录
3. 规则空白 → 生成宪法条款草案
4. 外部不可抗力 → 仅记录

### Phase 3: 宪法半自动草案生成
主Agent根据第3类结论自动生成宪法条款草案：
- 格式：一句话规则 + 适用角色 + 触发条件 + group
- 写入 `constitution/proposals.md`
- 不立即生效

### Phase 4: 用户确认 → 合并
用户确认后：
1. 将proposals.md中已确认的条款合并到full-constitution.md
2. 合并前按ID去重检查
3. proposals.md合并后清空

### Phase 5: 宪法瘦身提醒
- 超25条 → 自动提醒用户
- M01标记immutable，不可删除

## 注意
模块三完全内化到主Agent，不创建子Agent，保持注意力集中。

# 模块二 测试与质量门规则

> 测试闸门 + 审计闸门 + 质量维度覆盖
>
> **加载时机**：仅编排器 + QA Agent（code-qa、func-qa、audit）

| ID | 条款 |
|:---:|------|
| C27 | 测试文件存在闸门：源码文件必须存在对应测试文件，否则打回。 |
| C28 | 质量门双激活：GATE_1（debate-output.json schema 验证）+ GATE_2（任务完整性检查）在模块二启动时自动执行，不通过则打回模块一。 |
| C29 | 测试闸门：测试覆盖率≥70%或所有public函数均有测试用例。不通过打回开发Agent重写测试。 |
| C32 | 有效断言检查：每测试文件≥2个有效断言，func-qa负责验证。不满足拒绝通过。 |
| C33 | 审计闸门：代码必须经过audit Agent完整审计，无高危漏洞才可交付。 |
| C40 | 多维度质量门：QA检查应覆盖以下新维度 — ①错误韧性(retry/熔断/降级) ②API设计(RESTful/分页/版本/格式) ③文档质量(docstring/类型标注/示例) ④代码可维护性(DRY/解耦/平铺) ⑤多阶段管道健壮性。v12-v16实验证据：5维度A组4-16%→B组65-100%。 |

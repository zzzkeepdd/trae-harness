# Phase 0 专用规则

> 复杂度分级 + 攻击策略配置

| ID | 条款 |
|:---:|------|
| C01 | 用户可见产品=是 → 至少L2。任何网页/App/GUI都不可走L1。 |
| C02 | L1判据必须全部满足：≤3文件、单一技术栈、无用户可见产品、无外部依赖。 |
| C03 | L3判据任一满足即L3：文件数>15、外部API≥3个、复杂并发/多角色权限、3种+技术栈。 |
| C04 | 其他情况=L2。 |
| C26 | 攻击数量门槛：L1 ≥1个≥3分且≥3个 | L2 ≥2个≥3分且≥4个 | L3 ≥3个≥3分且≥5个。 |
| C34 | 攻击维度池（v5-v11扩展）：性能、安全、规范、鲁棒性、并发、可维护性、API设计、文档、错误韧性、管道健壮性。must_fix清单中每个条目必须在attacks中有≥1条≥3分的对应攻击。 |

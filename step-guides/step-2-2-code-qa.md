# Step 2: 代码技术验收

## 你的任务
执行代码技术验收

## 规则（4条）

1. **验收顺序**
   - 集成测试
   - 静态检查
   - 安全扫描

2. **实际执行（C30）**
   - 必须运行测试文件并记录结果（通过数/失败数）
   - 必须运行 lint 工具并记录结果
   - 不能仅通过代码审查产出报告

3. **只验收当前代码**
   - 基于步骤1产出的代码
   - 不关注其他任务

4. **产出报告**
   - 《代码技术验收报告》
   - 必须包含：测试执行结果摘要、lint检查结果
   - 通过/失败状态

## 输入
- 代码文件路径
- 测试文件路径
- 验收标准

## 产出
- 《代码技术验收报告》（含执行证据）
- 通过/失败状态

## 你不知道
- 前面开发的具体细节
- 后面有func-qa
- 如果失败后的整改流程
